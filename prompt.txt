🔐 前提：
Firebaseコンソールで表示される「ウェブ API キー」は、Google Cloud Consoleで作成・管理されているAPIキーと同一のものです。

🔁 キーの生成と同期：
Firebaseプロジェクトを作成すると、自動的に Google Cloud 側で APIキーが生成されます。

Firebaseの「プロジェクト設定」に表示されるAPIキーは、そのプロジェクトに紐づく 「現在有効なAPIキー」 を参照しています。

過去に存在していた古いAPIキー（自動生成含む）を 手動で削除した場合、Firebaseに表示されるAPIキーも 自動的に最新のものへ切り替わります。

✅ 今回の状況：
以前使っていたAPIキーをGoogle Cloud Consoleで削除済み。

Firebaseコンソールに表示されている「ウェブ API キー」は、削除後に自動的に切り替わった最新のAPIキーであることが確認できています。

.env.local および Vercel の環境変数にも、この最新のキーを反映済みです。

💡 結論：
Firebaseの「プロジェクト設定」で表示されるAPIキーは、Google Cloudで現在有効なキーと一致しており、正しく運用可能なものです。
したがって、.env.local に記載されている NEXT_PUBLIC_FIREBASE_API_KEY も現在のもので問題ありません。
